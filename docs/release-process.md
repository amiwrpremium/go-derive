# Release process

Fully automated. You should never tag manually.

## How it works

1. PRs merge to `master` with [Conventional Commits](https://www.conventionalcommits.org/)
   in their messages.
2. [release-please](https://github.com/googleapis/release-please) reads
   the commit log on `master` and opens (or updates) a
   `chore: release X.Y.Z` PR with:
   - the bumped version (computed from commit types: `feat` → minor,
     `fix` → patch, `BREAKING CHANGE:` → major)
   - the changelog entries it would generate
   - an updated `derive.go:Version` constant
3. **Merging the release PR** cuts the GitHub release, tags `vX.Y.Z`,
   and triggers `release.yml`.
4. `release.yml` attaches the **SBOM**, **cosign signature**, and **SLSA
   Level 3 provenance** to the GitHub release.

## Conventional Commits

| Commit type | Section in CHANGELOG | Bump |
|---|---|---|
| `feat:` | Features | minor |
| `fix:` | Bug Fixes | patch |
| `perf:` | Performance Improvements | patch |
| `deps:` | Dependencies | patch |
| `revert:` | Reverts | patch |
| `docs:` | Documentation | (none) |
| `refactor:` | Code Refactoring | (none) |
| `test:` | Tests | (none) |
| `build:` | Build System | (none) |
| `ci:` | Continuous Integration | (none) |
| `chore:` | Miscellaneous | (none) |

Append `!` after the type (e.g. `feat!:`) or include
`BREAKING CHANGE: <description>` in the body to force a major bump.

Examples:

```text
feat(rest): add GetFundingRateHistory

fix(ws): drain pending RPCs on connection close

deps: bump go-ethereum to v1.17.3

feat(auth)!: rename Signer.Sign to Signer.SignAction

BREAKING CHANGE: rename to disambiguate from header signing.
```

## What the release artifacts contain

When a release is published, three files are attached:

### 1. `go-derive-sbom.cdx.json` — CycloneDX SBOM

Generated by `anchore/sbom-action`. Lists every direct and transitive
dependency that the release was built with, including their licenses and
PURLs.

### 2. `go-derive-sbom.cdx.json.cosign-bundle` — cosign signature

[Cosign](https://github.com/sigstore/cosign) keyless OIDC signature over
the SBOM, anchored to the GitHub Actions identity that produced it.
Verify with:

```bash
cosign verify-blob \
    --bundle go-derive-sbom.cdx.json.cosign-bundle \
    --certificate-identity-regexp 'https://github.com/amiwrpremium/go-derive/.+' \
    --certificate-oidc-issuer 'https://token.actions.githubusercontent.com' \
    go-derive-sbom.cdx.json
```

### 3. `go-derive.intoto.jsonl` — SLSA Level 3 provenance

In-toto attestation produced by the
[SLSA generic generator](https://github.com/slsa-framework/slsa-github-generator).
Cryptographically proves which Actions workflow produced which artifact.
Verify with the
[slsa-verifier](https://github.com/slsa-framework/slsa-verifier).

## Configuration

Two files drive release-please:

| File | Purpose |
|---|---|
| [`.github/release-please-config.json`](../.github/release-please-config.json) | release type (`go`), changelog sections, the `derive.go` extra-file rule |
| [`.github/.release-please-manifest.json`](../.github/.release-please-manifest.json) | current version (release-please writes this) |

Both are committed; release-please updates the manifest as part of the
release PR.

## Manually retrying a release flow

If the `release.yml` workflow fails (e.g. transient SLSA generator
issue), redispatch it manually:

```text
GitHub UI → Actions → release → Run workflow → tag: vX.Y.Z
```

The job is idempotent: already-uploaded artifacts are clobbered.

## Versioning

The SDK is at `0.x` while the API surface is still evolving. Per Go's
module convention, breaking changes between `0.x` versions are allowed
but flagged via `BREAKING CHANGE:` so release-please bumps the minor
component.

We'll go to `1.0.0` once the API is committed for downstream stability.
After that, breaking changes will require either a `feat!:` or a v2
module path (`/v2`).

## What you have to do as a maintainer

Two things, only when the release-please PR is open:

1. **Review the changelog** in the auto-generated PR.
2. **Merge the PR**.

That's it. Everything else is automated.
